Day 1: File System MechanicsStep 1: Linux

Master Linux directory trees and navigation pathways.

📚 Deep Dive Tutorial:
The Linux file system is an inverted tree starting at root (/). Key paths include /etc for core configurations, /var/log for systems logging logs, and /home for individual user workspace scopes.
🛠️ Hands-on Lab:
Create nested play spaces via terminal:
mkdir -p ~/devops/linux_labs && cd ~/devops/linux_labs && pwd
Day 2: Text Processing StreamsStep 1: Linux

Study stream pipes, text matching regular filters, and outputs extraction lines.

📚 Deep Dive Tutorial:
Pipes (|) route the output of one command to the input of another. Use grep to look for string matches, awk to isolate data columns, and sed to substitute configurations elements on the fly.
🛠️ Hands-on Lab:
Filter out HTTP server logs errors streams:
cat log.txt | grep "500"
Day 3: Permissions and Task MonitoringStep 1: Linux

Understand user execute access limits and check resource metrics tracking channels.

📚 Deep Dive Tutorial:
Linux uses bitmasks (Read=4, Write=2, Execute=1) mapped across Owner, Group, and Public scopes. Control file safety levels via chmod, and check system resource stress pools running commands like top or ps aux.
🛠️ Hands-on Lab:
Write a script file, modify access flags, and execute:
chmod 755 test.sh && ./test.sh
